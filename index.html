⟩ HOW IT DETECTS

Pattern matching, not guesswork.

STRYX uses a rule-based heuristic engine — no AI, no external APIs. Every URL and email is checked against a set of known threat patterns, each with a risk score. The scores add up to give a final verdict from 0 to 100.

⟩

Regex Pattern Matching

Each rule uses a regular expression to detect specific patterns — like character substitutions (paypa1), suspicious keywords (/verify), or dangerous TLDs (.xyz). If matched, a score is added.

⟩

Cumulative Risk Scoring

Every matched rule adds to the total score (max 100). The more patterns match, the higher the score. Score determines the verdict: SAFE → LOW → MEDIUM → HIGH → CRITICAL.

⟩

Severity Levels

Each rule has its own severity — CRITICAL, HIGH, MEDIUM, or LOW. A single CRITICAL match (like a spoofed sender) can push the score very high even without other indicators.

⟩

MITRE ATT&CK Mapping

The highest-severity indicator is mapped to a real MITRE ATT&CK technique ID — the same framework used by professional SOC teams worldwide to classify attacks.

⟩

Zero External Calls

No data leaves your browser. No VirusTotal, no threat intel API, no server. All 20+ rules run locally in JavaScript — making STRYX instant, private, and always available offline.

⟩

Limitations

Rule-based detection can produce false positives. STRYX is a first-pass triage tool — always verify findings with additional investigation before taking major action.

About STRYX

The story, the mission, the builder.

WHAT IS STRYX

STRYX is an open-source email and URL threat analyzer built for SOC analysts, cybersecurity students, and anyone who wants to verify suspicious links and emails before clicking.

It uses a custom-built heuristic rule engine with 20+ detection rules — no AI, no external APIs, no subscription. Everything runs locally in your browser. Your data never leaves your device.

STRYX maps every finding to the MITRE ATT&CK framework — the same standard used by professional security teams worldwide.
